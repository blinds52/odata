5.1.2 ODATA-924 - Define semantics for PUT against a single entity with nested content

Ram: ODATA-924 is OPEN, no objections

Mike: Discussion: Semantics should align with POST;
1) You can include references (only) to existing entities (no content)
2) The end result of the PUT is that the existing (references) and new (entities) are the full set of entities related according to that relationship.

Mike: Define that, when you do a PUT,
1) You can include new entities and references to existing entities, but not content for existing entities
2) The end result of the PUT is that the existing (references) and new (entities) are the full set of entities related according to that relationship.

Mike: I move we resolve ODATA-924 as proposed. Mark seconds.

No objections, issue resolved as proposed. ODATA-924 is RESOLVED as proposed.

Ram: Back from ODATA-927 (cf. section 5.1.3 below)

Ram: No objection to opening the discussion on the proposal for ODATA-924.

Ram: The intention is to revise the previous resolution.

Mike: Revised proposal to align with PATCH (as per ODATA-876)

Mike: Define that, when you do a PUT,
1) You can include new entities, existing entities, and references to existing entities
2) Existing entities are updated using PUT semantics
a) entities to be updated must have either @odata.id or key fields, otherwise interpreted as an insert
3) The end result of the PUT is that the specified entities and entity references are the full set of entities related according to that relationship.

Ralf: From 11.4.3: Key and other non-updatable properties, as well as dependent properties that are not tied to key properties of the principal entity, can be omitted from the request. If the request contains a value for one of these properties, the service MUST ignore that value when applying the update.

Mike: Revised proposal for ODATA-924, to align with previously approved ODATA-876:
Align Semantics of PUT with PATCH, as specified in ODATA-876, by specifying that, when you do a PUT,
1) You can include new nested entities, existing entities, and references to existing entities
2) Existing entities are updated using PUT semantics
a) entities to be updated must have either @odata.id or key fields, otherwise interpreted as an insert
b) if @odata.id and key fields both exist but don't match, then request is invalid
3) The end result of the PUT is that the specified entities and entity references are the full set of entities related according to that relationship.

Mike: I move to revise the previously approved resolution to ODATA-924 as described in the revised proposal. Matt seconds.

No objections, the motion passes with unanimous support. ODATA-924 is RESOLVED as proposed.
