6.2.1 Discussion
  • Stefan recommends the draft document
  • Martin asks about cost to get access to OData status
  • Ed REST APIs possible competitors to OData
  • Mark: Oasis Standard should still be downloaded with no fee even if ISO charges a fee
  • Hubert states, that OData provides a superset of protocol capabilities
  • ED assumes the possibility of a possible bloat view of OData
  • Ram and ED suggest to evangilize protocol included in open source community
  • All suggest, to remind folks of discussion about using ODBC: Argument of high standard conformance benefits versus low tech efficiency
  • All also remind, that OData allows for different levels of compliance. For instance level mininal allows for low effort compliance as a head start in reaching higher levels of conformance where everyone will benefit from (interoperability etc.)
  • Mike and Hubert further emphasize interoperability; OData encapsulates best practises of software engineering
  • Ed consider Node.js /server as possibly breaking OData expectations
  • Hubert mentions server side library and client library as scope already accomplished/covered by OData
  • All agree on a deeper discussion for a future agenda.
  • Ted suggests to consider comparison articles on protocols and exploring REST APIs (virt.x)
  • Ram asks all to look at Node.js and virt.x
  • Ed adds to also look at backbone.sync (popular javascript framework with a CRUD rest method.

6.3 Approval to start a Special Majority Ballot to approve submitting OData v4 and OData JSON Format v4 for international standardization

Hubert: I move that the Chair request the TC admin to start a Special Majority Ballot to approve submitting the request (https://www.oasis-open.org/apps/org/workgroup/odata/download.php/53050/Submission%20request%20for%20international%20standardization.docx) to the OASIS Board president to advance OData v4.0 and OData JSON Format v4.0 OASIS Standards incorporating OASIS Approved Errata to an International Standard via ISO/IEC Joint Technical Committee 1 per the OASIS Liaison Policy (https://www.oasis-open.org/policies-guidelines/liaison).

Mark seconds

No discussion, no objections, motion passed, unanimous consent
