6.1.2 ODATA-747 - Allow IEEE 754 constants [-]INF and NaN also for Edm.Decimal and integer types

Ram: ODATA-747 is OPEN, no objections

Gerald: is ok with restricting ODATA-747 to Edm.Decimal

Mike: Question is: what should server do for 4.0 clients?
Some Options:

1) return null
    a) perhaps with an annotation to specify that it's +/- INF or NaN?

2) use a different type
3) server error 500
4) return the string +/- INF or NaN and give the client the chance figure it out
5) return any value, along with the instance annotation (i.e., for a property that is not nullable)

Ram: Let's research how different programming languages deal with +/- INF and NaN

Mike: It may help to understand what client programming languages do in their decimal type equivalent.

Mike: If the generated code would have to do something different in order to accept a value of +/-INF and NaN, then we should probably add a metadata annotation...
