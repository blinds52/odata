5.1.2 ODATA-751 "Clarification of Edm.NavigationPropertyPath and Edm.PropertyPath"

Ralf: Last change to path syntax is not fully reflected in specification text.
Ralf: Change in 14.5.13 the sentence "In contrast to the edmath expression, the value of the edmropertyPath expression is the path itself, not the value of the property identified by the path." to "In contrast to the edmath expression, the value of the edmropertyPath expression is the path itself, not the value of the property or the value of the term cast identified by the path."
Ralf: Change in 14.5.11 "In contrast to the edmath expression, the value of the edm:NavigationPropertyPath expression is the path itself, not the target instance(s) of the navigation property identified by the path." to "In contrast to the edmath expression, the value of the edm:NavigationPropertyPath expression is the path itself, not the instance(s) identified by the path."
Ralf: The last path segment MUST resolve to a navigation property in the context of the preceding path part, or to a term cast where the term MUST be of type Edm.EntityType, a concrete entity type or a collection of Edm.EntityType or concrete entity type.

Ralf: ODATA-751 is open

Hubert: I move to resolve ODATA-751 as proposed. Martin seconds

Motion passes, no objections
