Notes on Structure of Display and Symbols Used

We divide all issues into active and closed issues. The active issues are displayed first, while the closed ones are displayed second.

Inside each of the two main sections, there are four tables. These tables map each issues processing stages as documented in meeting minutes to the corresponding section of the publicly accessible meeting minute document on the OASIS servers. Each issue is mapped into one of the four tables of its main section as derived from the types and priorities offered by JIRA:

  1. Class 1 Issues: Issue Type ≥ Bug and Priority ≥ Major
  2. Class 2 Issues: Issue Type like 1. but Priority < Major
  3. Class 3 Issues: Issue Type < Bug and Priority ≥ Major
  4. Class 4 Issues: Issue Type like 3. but Priority < Major

The tooltips on the inner cells assist in indicating the date of the meeting. The link targets of the inner cells point to the relevant subsection in the meeting document. the summary row at the bottom of each table yields the count of issues processed during that meeting and is linked with the meeting document. Each row can be addressed individually by appending a fragment like eg "#odata-123" to the URL of this document. The background color of the leftmost columns cell w.r.t. Issue-IDs somehow encodes the status as received from JIRa and is as follows: New, Open, Proposed, Resolved, Applied, Closed. Hovering with the mouse over such a cell also triggers a tooltip stating useful metrics as retrieved from JIRA. This document is updated after each meeting and after approval of the relevant minute document by the TC and is provided in the hope, it is useful.
