5.1 OData CSDL

5.1.1 ODATA-801 "Consider supporting negative scale for decimal type"

ODATA-801 is OPEN

Mike: changing the definition of Scale might break clients

Mike: pretending a precision larger than actually stored might result in loss of precision, but clients will be able to read the data

Mike: could add an annotation that tells that the actual precision and scale

Mike: I propose we defer ODATA-801 to 4.1 release. In 4.0, services should add the absolute value of the negative scale to the returned precision. Add note to spec advising clients to be aware in this case that rounding may occur. Martin seconds.

ODATA-801 is DEFERRED to 4.1, will add note to 4.0 spec

5.2 OData JSON Format

5.2.1 ODATA-771 "Exponential notation for Edm.Decimal values in JSON payloads"

Ralf: Olingo Java client rejects exponential notation

Ralf: NewtonSoft JSON library converts values with exponential notation as double, which then might lead to problems with the layer on top

Ralf: C# library also treats it as a double and rejects it because it isn't a decimal

Ralf: Way forward could be to add a format parameter for the JSON format that clients can specify if they accept exponential notation for decimals

Mike: two options:

  • preference in Prefer header (Protocol, applies to all formats)
  • format parameter (specific to JSON)

Mike: seems to be more format-related than protocol-related

Ralf: DecimalWithExponentialNotation=true

Ralf: No problem

Ralf: DecimalExponentialNotation=true

Ralf: ExponentialDecimals=true

Ralf: ScientificDecimals=true

Ralf: E-Decimal

Ralf: ExpDecimal

Mike: suggest: add a format parameter, "ExponentialDecimals", that tells the service that the client is able to receive decimals as exponential notation. In presence of this format parameter, services MAY (but are not required to) persist decimal values in exponential notation.

Mike: Clients should verify in Capabilities.SupportedFormats whether or not service supports this before sending decimal values in exponential notation.

Hubert: I move to resolve ODATA-771 by means of introducing a format parameter, named "ExponentialDecimals", that tells the service that the client is able to receive decimals using exponential notation. In presence of this format parameter, services MAY (but are not required to) serialize decimal values in exponential notation. Mike seconds.

ODATA-771 is resolved as proposed
