5.7 ODATA-702 "Define the behaviour of system query option $count and path segment /$count in combination with $apply"

Ralf:

$apply on resource paths ending in /$count acts on the set "preceding" the /$count, the result is the plain-text number of items in the result of $apply, similar to the combination of /$count and $filter

$count system query option is executed after $apply, @odata.count contains the number of items in the result of $apply

ODATA-702 is OPEN

Hubert: I move to resolve ODATA-702 as proposed. Stefan seconds.

ODATA-702 is resolved as proposed

5.9 ODATA-662 "Allow usage of $apply as "Expand Option" in $expand"

Martin: We haven't specified the usage of $apply as "Expand Option" in $expand. There are good use cases for $apply as an option of $expand (and it combines very well):

GET Products?$expand=Sales($apply=aggregate(Amount with sum as Total))

Returns all products, and the Sales navigation property value is a one-element array with the Total Means: per Product in top-level result inline result of

Mike: does the syntax of $apply prevent nesting it in $expand?

Martin: no, no semicolons used in $apply syntax

Mike: seems like a pretty core thing

ODATA-662 is OPEN

Mike: I move to resolve ODATA-662 as proposed. Gerald seconds.
